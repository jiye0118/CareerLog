  ---
  GET이 뭔지

  브라우저가 서버한테 "이 페이지 줘!" 하고 요청하는 방식이에요.

  브라우저 → 서버: "GET /login" (= /login 페이지 내놔)
  서버 → 브라우저: "200 OK, 여기 있어" (= 성공적으로 줬다)

  숫자 200은 "성공"이라는 뜻이에요. 마치 택배가 정상 배달됐을 때 "배송완료" 같은
  거예요.

  ---
  왜 두 번 나오는지

  첫 번째 줄이랑 두 번째 줄 시간을 봐요.

  첫 번째: 8.0s  ← 8초
  두 번째: 258ms ← 0.25초

  첫 번째 — 브라우저에서 처음 /login을 열었을 때예요. Next.js가 이 페이지를 처음으로
   컴파일하느라 8초 걸린 거예요. 코드를 처음 읽고 번역하는 시간이에요.

  두 번째 — 브라우저가 페이지를 다 받은 뒤 내부적으로 한 번 더 확인 요청을 보낸
  거예요. 이미 컴파일된 결과물이 있어서 0.25초로 훨씬 빠르게 응답했어요.
