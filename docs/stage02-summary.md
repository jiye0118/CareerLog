# Stage 02 작업 요약

## 완료된 작업

**새로 생성된 파일:**

| 파일 | 역할 |
|------|------|
| `middleware.ts` | 세션 갱신 + `/dashboard` → `/login` 리다이렉트 보호 |
| `app/(auth)/login/actions.ts` | `signIn` / `signUp` Server Actions |
| `app/(auth)/login/page.tsx` | 로그인·회원가입 탭 UI (클라이언트 컴포넌트) |
| `app/dashboard/actions.ts` | `signOut` Server Action |
| `app/dashboard/layout.tsx` | 헤더(앱 이름 + 로그아웃 버튼) + 인증 체크 |
| `app/dashboard/page.tsx` | 기록 목록 (서버 컴포넌트) |
| `app/dashboard/components/LogForm.tsx` | 업무 기록 입력 폼 + AI 분석 결과 표시 |

**업데이트된 파일:**
- `app/page.tsx` — `/dashboard`로 리다이렉트
- `app/layout.tsx` — 메타데이터를 CareerLog로 변경

**동작 흐름:** 미인증 상태로 `/` 접근 → `/dashboard` → 미들웨어가 `/login`으로 리다이렉트 → 로그인 후 `/dashboard` → 기록 입력 → AI 분석 결과 표시 + 목록 갱신(`router.refresh()`)

---

## 각 파일 상세 설명

### 전체 그림부터 이해하기

웹사이트를 만들 때 크게 두 가지가 필요해요.

- **화면** — 사용자 눈에 보이는 것 (버튼, 폼, 글자)
- **문지기** — 화면을 보여주기 전에 "넌 누구야?" 확인하는 것

Stage 02에서 만든 건 바로 이 두 가지예요.

---

### 1. `middleware.ts` — 아파트 경비원

```
사용자가 주소창에 /dashboard 입력
        ↓
  경비원이 가로막음
  "로그인 했어요?" 확인
        ↓
  YES → 통과, 대시보드로 이동
  NO  → 되돌려 보냄, 로그인 페이지로 이동
```

**미들웨어**란 사용자가 어떤 페이지를 열기 **전에** 무조건 실행되는 코드예요.

경비원처럼 모든 방문자를 일일이 확인해요:
- `/dashboard` 가려는데 로그인 안 됨 → `/login`으로 돌려보냄
- `/login` 가려는데 이미 로그인 됨 → `/dashboard`로 보내버림 (이미 들어왔으니까)

---

### 2. `app/(auth)/login/` — 로그인·회원가입 페이지

**3개 파일**로 이루어져 있어요.

#### `actions.ts` — 실제로 일하는 직원 (서버에서 실행)

```
사용자가 폼을 제출하면 이 코드가 실행됨

signIn  → 이메일+비밀번호로 Supabase에 "이 사람 맞아요?" 확인 요청
          → 맞으면 /dashboard로 이동
          → 틀리면 "비밀번호가 틀렸어요" 메시지 반환

signUp  → Supabase에 새 계정 만들어달라고 요청
          → 완료되면 "확인 이메일 발송" 메시지 반환
```

이 코드는 **서버에서만** 실행돼요. 왜냐면 비밀번호 같은 민감한 정보를 다루기 때문이에요. 사용자 브라우저에서 실행되면 해킹당할 수 있거든요.

#### `page.tsx` — 화면에 보이는 것 (브라우저에서 실행)

```
┌─────────────────────────────┐
│         CareerLog           │
│     커리어 기록을 시작하세요     │
│                             │
│  [ 로그인 ]  [ 회원가입 ]      ← 탭 전환
│                             │
│  이메일: [____________]      │
│  비밀번호: [____________]    │
│                             │
│       [ 로그인 ]             │
└─────────────────────────────┘
```

탭을 누르면 로그인/회원가입 폼이 바뀌어요. 버튼을 누르면 위의 `actions.ts` 직원에게 일을 시켜요.

---

### 3. `app/dashboard/` — 로그인 후 보이는 메인 화면

#### `layout.tsx` — 모든 대시보드 페이지의 공통 틀

```
┌─────────────────────────────────┐
│  CareerLog            로그아웃   │  ← 이 헤더가 layout
├─────────────────────────────────┤
│                                 │
│    (여기에 각 페이지 내용)         │  ← 이 안에 page.tsx가 들어감
│                                 │
└─────────────────────────────────┘
```

레이아웃은 **여러 페이지가 공통으로 쓰는 껍데기**예요. 헤더를 한 번만 만들어두면 대시보드 안의 모든 페이지에 자동으로 붙어요.

그리고 여기서도 한 번 더 "로그인 됐어?" 확인해요. 혹시 미들웨어를 뚫고 들어온 사람이 있을 경우를 대비해서요.

#### `actions.ts` — 로그아웃 담당

```
로그아웃 버튼 클릭
      ↓
Supabase에 "이 사람 세션 끊어줘" 요청
      ↓
/login 페이지로 이동
```

딱 한 줄짜리 일이지만, 이것도 **서버에서 실행**돼야 해요. 세션 쿠키를 서버에서 삭제해야 하거든요.

#### `components/LogForm.tsx` — 업무 기록 입력 폼

```
┌──────────────────────────────────┐
│  오늘의 업무 기록                  │
│                                  │
│  [직접 입력 ▼]                    │
│                                  │
│  ┌────────────────────────────┐  │
│  │ 오늘 결제 API 성능 개선...   │  │
│  │                            │  │
│  └────────────────────────────┘  │
│                      [기록 저장]  │
│                                  │
│  --- AI 분석 완료 ---             │  ← 저장 후 나타남
│  Redis 캐싱을 도입하여 응답속도    │
│  800ms → 200ms 개선              │
│  [Redis] [캐싱] [성능최적화]       │
└──────────────────────────────────┘
```

이건 **브라우저에서 실행**돼요. 왜냐면 사용자가 글자를 입력할 때마다 화면이 바뀌어야 하거든요 (타이핑할 때 실시간으로 보여야 하니까).

"기록 저장" 버튼을 누르면:
1. Stage 01에서 만들어둔 `/api/intake`에 기록을 보냄
2. 서버에서 Claude AI가 분석
3. DB에 저장
4. 분석 결과를 폼 아래에 보여줌
5. 아래쪽 기록 목록을 새로고침

#### `page.tsx` — 기록 목록

```
┌──────────────────────────────────┐
│  최근 기록                        │
│                                  │
│  ┌────────────────────────────┐  │
│  │  2026-04-22       [backend]│  │
│  │  Redis 캐싱 도입으로 API    │  │  ← AI가 요약한 내용
│  │  응답속도 75% 개선           │  │
│  │  [Redis] [캐싱] [성능최적화] │  │
│  └────────────────────────────┘  │
│                                  │
│  ┌────────────────────────────┐  │
│  │  2026-04-21    [leadership]│  │
│  │  ...                       │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

이건 **서버에서 실행**돼요. 페이지가 열릴 때 Supabase DB에서 내 기록들을 가져와서 보여줘요.

---

### 서버 vs 브라우저, 왜 나눌까?

| | 서버에서 실행 | 브라우저에서 실행 |
|---|---|---|
| **예시** | 로그인, 로그아웃, 기록 목록 조회 | 타이핑, 탭 전환, 버튼 클릭 |
| **이유** | DB 비밀번호·API 키가 있어서 노출되면 안 됨 | 사용자 입력에 실시간 반응해야 함 |

---

### 전체 흐름 한눈에

```
사용자 접속
    ↓
middleware.ts가 확인
    ↓
로그인 안 됨 → /login 페이지
    ↓
이메일+비밀번호 입력 → actions.ts가 Supabase에 확인
    ↓
로그인 성공 → /dashboard
    ↓
layout.tsx 헤더 표시
    ↓
page.tsx에서 기록 목록 불러오기
LogForm.tsx 입력 폼 표시
    ↓
업무 기록 입력 → /api/intake → Claude AI 분석 → DB 저장
    ↓
분석 결과 표시 + 목록 새로고침
```
